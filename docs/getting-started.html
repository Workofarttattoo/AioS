Getting Started

Your journey into defensive security begins here

‚ö†Ô∏è Important: These tools are designed for authorized security assessments only. You MUST have explicit permission before testing any system you don't own. Unauthorized use is illegal and unethical.

1

Clone the Repository

First, clone the Ai:oS repository which includes the Red Team Tools Suite:

# Clone via HTTPS git clone https://github.com/Workofarttattoo/AioS.git cd AioS # Or clone via SSH git clone git@github.com:Workofarttattoo/AioS.git cd AioS

‚ÑπÔ∏è Note on Encrypted Files: Some files are encrypted with git-crypt. You'll see the public interfaces and documentation, but encrypted source code will appear as binary data. This protects proprietary algorithms while keeping usage patterns open.

2

Install Dependencies

Install the required Python packages:

# Create a virtual environment (recommended) python3 -m venv venv source venv/bin/activate # On Windows: venv\Scripts\activate # Install base dependencies pip install numpy scipy # Optional: Install ML/Quantum dependencies pip install torch # Optional: Install development dependencies pip install pytest black mypy

‚úÖ Lightweight Setup: The red-team-tools suite has minimal dependencies. Most tools only require NumPy and SciPy!

3

Verify Installation

Run health checks to ensure everything is working:

# Check AuroraScan python -m red-team-tools.aurorascan --health --json # Check CipherSpear python -m red-team-tools.cipherspear --health --json # Check MythicKey python -m red-team-tools.mythickey --health --json

You should see JSON output indicating the health status of each tool.

4

Run Your First Tool in Demo Mode

Try running a tool in safe demo mode (no actual network activity):

# Run AuroraScan in demo mode python -m red-team-tools.aurorascan --demo --json # Run CipherSpear in demo mode python -m red-team-tools.cipherspear --demo --json # Run MythicKey in demo mode python -m red-team-tools.mythickey --demo --profile gpu-balanced

Demo mode simulates tool behavior without making actual connections or tests.

7

Best Practices

Follow these best practices for safe and effective use:

  • Always get authorization before testing any system
  • Start with --demo mode to understand tool behavior
  • Use --health checks before production use
  • Test in isolated environments (VMs, lab networks)
  • Read tool documentation before running scans
  • Respect rate limits to avoid disruption
  • Log all activities for audit trails
  • Report findings responsibly via proper channels
